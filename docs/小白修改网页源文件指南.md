# 个人网站小白修改手册

这份手册是给“不懂代码，但想自己改网站内容”的你准备的。

你可以把这个网站先理解成一句话：

- `app/` 里放的是“每个页面写什么”
- `components/` 里放的是“页面里反复出现的小模块”
- `lib/` 里放的是“项目数据”
- `app/globals.css` 里放的是“全站长什么样”

---

## 先记住两个网址

- 首页：`http://localhost:3000/`
- Life 页面：`http://localhost:3000/life`

如果你改的是 `app/life/page.tsx`，就一定要去看 `/life`，不是看首页。

---

## 最常改的地方

### 改首页标题和首页文案

改这里：

- `app/page.tsx`
- `components/HeroSection.tsx`

你会在里面看到类似这些文字：

- `用工程师的眼睛`
- `用 AI 做实验`
- `工程背景，数据驱动，正在用 AI 重建我看世界的方式。`

适合改什么：

- 首页大标题
- 首页副标题
- 首页“Who I am”那一段介绍

---

### 改导航栏

改这里：

- `components/Navbar.tsx`

你会看到一组像这样的内容：

```tsx
const navLinks = [
  { href: "/", label: "Home" },
  { href: "/projects", label: "Projects" },
  { href: "/about", label: "About" },
  { href: "/lab", label: "Lab" },
  { href: "/life", label: "Life" },
];
```

适合改什么：

- 导航名称，比如把 `Home` 改成 `首页`
- 导航顺序

不建议你现在就改什么：

- `href`

因为 `href` 是跳转地址，改错了会点不开页面。

---

### 改 About 页面

改这里：

- `app/about/page.tsx`

适合改什么：

- 自我介绍
- 你的背景
- 为什么做 AI
- 你的思考方式

这是最适合你自己慢慢改文案的页面之一。

---

### 改 Lab 页面

改这里：

- `app/lab/page.tsx`

适合改什么：

- “知识实验室”介绍
- “正在建设中”的提示文案

---

### 改 Life 页面

改这里：

- `app/life/page.tsx`

这个文件里有一组卡片数据，例如：

```tsx
const experiments = [
  {
    icon: "🍞",
    title: "Baking Lab",
    description: "研究面筋形成、发酵机制与温度控制——用工程师视角解构烘焙科学。",
    status: "探索中",
  },
];
```

适合改什么：

- `title`：卡片标题
- `description`：卡片说明
- `status`：右上角小状态
- `icon`：emoji 图标

举例：

- 把 `status: "探索中"` 改成 `status: "快乐进行中"`
- 改完后要看 `http://localhost:3000/life`

---

### 改项目列表和项目详情

改这里：

- `lib/projects.ts`

这是整个项目区最重要的数据文件。你会看到每个项目都长这样：

```tsx
{
  slug: "ai-workflow",
  title: "Personal AI Workflow",
  description: "构建个人知识管理和 AI 协作流程，让学习和创造效率翻倍。",
  category: "AI System",
  tools: ["ChatGPT", "Claude", "Obsidian", "Python"],
  year: 2026,
  status: "in-progress",
}
```

适合改什么：

- 项目标题 `title`
- 项目简介 `description`
- 工具列表 `tools`
- 年份 `year`
- 状态 `status`
- 项目详情中的背景、思考、成果、未来方向

你可以把它理解成：

- `app/projects/page.tsx` 决定“项目列表页怎么摆出来”
- `app/projects/[slug]/page.tsx` 决定“项目详情页怎么显示”
- `lib/projects.ts` 决定“项目内容本身是什么”

所以如果你想改“项目写了什么”，大多数时候改 `lib/projects.ts` 就够了。

---

### 改页脚

改这里：

- `components/Footer.tsx`

适合改什么：

- 底部一句话
- 品牌名称

---

### 改全站颜色和字体风格

改这里：

- `app/globals.css`

这里最值得关注的是这些颜色变量：

```css
--color-background
--color-foreground
--color-accent
--color-muted
--color-border
--color-card
```

你可以把它理解成：

- `background`：页面大背景色
- `foreground`：主要文字颜色
- `accent`：强调色，按钮和重点文字常用
- `card`：卡片背景色
- `border`：边框颜色

如果你想整体换风格，优先改这里，不要一个页面一个页面地乱改。

## 你暂时不要改的地方

这些文件先不要手动改：

- `.next/`
- `node_modules/`
- `package-lock.json`
- `next-env.d.ts`
- `tsconfig.json`
- `postcss.config.mjs`
- `eslint.config.mjs`
- `next.config.ts`

原因很简单：

- 它们不是网站内容
- 改错了可能导致网站起不来

## 为什么“我改了文件，网页没变化”

最常见有 3 个原因：

### 1. 改对了文件，但看错了页面

例如：

- 你改了 `app/life/page.tsx`
- 但你看的却是首页 `/`

那当然不会变。

---

### 2. 你现在跑的是“正式模式”，不是“编辑模式”

你电脑现在开机自动启动的网站，用的是正式模式。

正式模式的特点是：

- 比较稳定
- 适合平时直接打开看
- 但你改完文件，不会自动刷新

所以你每次改完内容后，需要这样让它更新：

```bash
cd "/Users/monster/Desktop/MONSTER/ai/个人网站/personal-lab"
npm run build
launchctl kickstart -k gui/$(id -u)/com.monster.personal-lab
```

---

### 3. 浏览器还是旧缓存

如果你明明已经重建了，还是没看到变化：

- 先刷新一次页面
- 不行就关闭这个标签页再重新打开

---

## 如果你想“边改边看”，最方便的方式

如果你接下来会连续改很多文案，推荐临时切到编辑模式：

```bash
launchctl bootout gui/$(id -u) ~/Library/LaunchAgents/com.monster.personal-lab.plist
cd "/Users/monster/Desktop/MONSTER/ai/个人网站/personal-lab"
npm run dev
```

然后打开：

- `http://localhost:3000/`

这时候特点是：

- 你保存文件后，网页通常会自动刷新
- 特别适合改标题、描述、状态、页面结构

改完以后，如果你想回到“开机自动打开”的稳定模式，再执行：

```bash
cd "/Users/monster/Desktop/MONSTER/ai/个人网站/personal-lab"
npm run build
launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/com.monster.personal-lab.plist
launchctl kickstart -k gui/$(id -u)/com.monster.personal-lab
```

## 最后给你的建议

如果你现在想自己安全地改内容，最推荐的顺序是：

1. 先改文案，不改结构
2. 先改 `app/` 和 `lib/`，不要动配置文件
3. 每次只改一小处，改完立刻看页面
4. 想长期连续修改时，用 `npm run dev`
